# Node Types (Triggers)

| Type | When to Use |
|------|-------------|
| `foundation` | Core concepts, axioms, starting points |
| `surprise` | Unexpected findings, contradictions to existing beliefs |
| `tension` | Conflicts, trade-offs, unresolved issues |
| `consequence` | Implications, downstream effects |
| `repetition` | Patterns that recur across contexts |
| `question` | Open questions, unknowns to explore |
| `decision` | Choice points, alternatives considered |
| `experiment` | Empirical tests, validation attempts |
| `analysis` | Agent-generated analysis (PURE reviews, critiques) |
| `serendipity` | Chaos-injected insights (from blind agent) |
| `randomness` | Raw random input before synthesis |
| `reference` | Single pointer to another project's node or external URL |
| `library` | Collection of references (bibliography section) |
| `thinking` | AI reasoning trace during chronological reading |
| `prediction` | Forward-looking belief ("I expect X to happen") |
| `evaluation` | Normative reflection ("This is good/bad/meaningful because...) |

---

## Metabolizing Information

Don't just "log" events. Metabolize raw inputs into structured understanding:

| When you feel... | Create Node | The Cognitive Act |
|------------------|-------------|-------------------|
| "This conflicts with X" | `tension` | **Synthesis**: Holding two opposing ideas |
| "Wait, that's weird..." | `surprise` | **Update**: Reality check against mental model |
| "So therefore..." | `consequence` | **Projection**: Foreseeing downstream effects |
| "I don't get X..." | `question` | **Curiosity**: Defining the boundary of understanding |
| "We've been here before" | `repetition` | **Pattern Recognition**: Identifying structural laws |
| "I choose X because..." | `decision` | **Agency**: Collapsing possibility into commitment |

---

## Connection Requirements

- **Only meaningful edges** - every edge must have a clear `why` you can articulate
- **Quality over quantity** - one well-explained edge beats three vague ones
- **Cross-link to existing knowledge** - connect your work to pre-flight nodes
- **Leave trails** - `question` nodes mark open threads for future agents

**Creating orphan nodes = failure. Creating duplicate concepts = failure.**
